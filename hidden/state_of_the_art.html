Modeling

2015 - Génevaux - Terrain Modeling from Feature Primitives

Introduce a hierarchical tree representation for 2D terrains composed of compact features primitives (noise-based mostly), and operators aggregating their subtrees. Inspired by the BlobTree and noise-based terrains introduced by Musgrave. The compact terrains can be visualized by either a high-quality mesh extraction or sphere tracing. This paper also shows some Lipschitz constant computation for several node in the tree.
Each terrain primitive comes with two functions: an elevation function and a weight function. The main difficulty resides in combining efficiently the primitives together and keeping tracks of the weight of every primitives or sub tree in the scene. Overall, a nice and efficient model which would benefit from an interactive editor to create custom heightmaps. Lots of work could be done on the primitives themselves to create interesting, parameterized features.
Pros:
-Can virtually represent any 2D features
-Compact, memory efficient model

Cons:
-Weight functions to combine primitives are difficult to control.
-Visualization is not straightforward - the model is an implicit tree.

Potentials:
-Interactive editor to place the primitives and create custom heightfields
-New primitives to represent different features, like coastlines.
